# Specification: Phase 2 - The Heart

## 1. Overview

**Track ID:** `heart_20260424`

**Description:** Implement Convex database with real-time agent sync for the Simulacra autonomous AI world.

**Goal:** Enable persistent agent data storage with real-time synchronization across all connected clients.

---

## 2. Functional Requirements

### 2.1 Convex Schema Deployment

Deploy the complete Convex schema as defined in `SCHEMA.md`:

**Core Tables:**
- `agents` - Agent data (name, archetype, position, needs, state)
- `world_state` - Global world state (weather, time, day count)
- `events` - Sensory buffer (last 10 events per agent)
- `config` - Master configuration (password hash, tick settings)

**Required Indexes:**
- `agents_by_active` - Filter active agents
- `agents_by_position` - Spatial queries
- `events_by_agent` - Agent event history
- `events_by_time` - Recent events

### 2.2 CRUD Operations

Implement basic CRUD operations for agents:

| Operation | Function | Description |
|-----------|----------|-------------|
| `getAll` | `agents.getAll()` | Retrieve all active agents |
| `getById` | `agents.get(id)` | Retrieve single agent by ID |
| `create` | `agents.create()` | Create new agent with archetype |
| `updatePosition` | `agents.updatePosition()` | Update agent grid coordinates |

**Agent Fields (from SCHEMA.md):**
- Identity: `name`, `archetype`
- Position: `gridX`, `gridY`
- Visual: `spriteVariant`
- State: `currentAction`, `targetX`, `targetY`
- Needs: `hunger`, `energy`, `social` (0-100)
- Status: `isActive`, `lastActiveAt`

### 2.3 Real-time Synchronization

**Implementation:** Convex useQuery for automatic real-time sync.

**Flow:**
1. Database mutation occurs (Cron or user action)
2. TanStack Start's `useQuery` receives updated agent array
3. `GameWorld` component detects data change
4. Excalibur Actor target coordinates updated
5. Smooth interpolation to new position

**Verification:** Opening two browser tabs should show synchronized agent positions.

### 2.4 World State Features

Implement world state management:

**Weather System:**
- States: sunny, cloudy, rainy, stormy
- Stored in `world_state` table
- Can be changed via Master Panel (Phase 5)

**Time Tracking:**
- `timeOfDay`: 0-24 hours (number)
- `dayCount`: Simulated days elapsed
- `tickIntervalSeconds`: 60-120 seconds (configurable)

**Initial State:**
- Default weather: sunny
- Default time: 12:00 (noon)
- Day count starts at 0

### 2.5 Agent Seed Data

Create 10-15 agents for MVP testing:

**Requirements:**
- Random names from archetype pools
- Random positions across 64×64 grid
- Initial needs: hunger=50, energy=50, social=50
- Random sprite variants (0-3)
- Mix of all 5 archetypes (builder, socialite, philosopher, explorer, nurturer)

**Seeding Process:**
1. Check if agents exist (skip if already seeded)
2. Generate agents with varied archetypes
3. Insert into Convex `agents` table
4. Log seeding completion

## 3. Non-Functional Requirements

### 3.1 Performance
- Real-time sync latency: < 100ms (Convex handles this)
- Query performance: < 50ms for agent list
- Max agents: 50 (without spatial partitioning)

### 3.2 Data Integrity
- All mutations validated server-side
- Position updates bounded to grid (0-63)
- Agent IDs auto-generated by Convex

### 3.3 Code Quality
- TypeScript types for all Convex functions
- Unit tests for CRUD operations
- >80% code coverage for new code
